## 한글의 디지털 표현: 핵심 기술

<CardGrid stagger>
  <Card title="문자 인코딩: 한글의 디지털 옷" icon="globe">
    컴퓨터는 모든 정보를 0과 1, 즉 이진수로 표현합니다. 한글도 예외는 아닙니다. 한글을 컴퓨터가 이해할 수 있는 형태로 변환하는 과정을 **문자 인코딩**이라고 합니다. 대표적인 한글 인코딩 방식으로는 **UTF-8**, **EUC-KR**, **CP949** 등이 있습니다.
    <Tabs>
        <TabItem label="UTF-8">
             전 세계 대부분의 문자를 표현할 수 있는 국제 표준 인코딩입니다. 한글은 주로 3바이트로 표현되며, 웹 환경에서 가장 널리 사용됩니다. 가변 길이 인코딩 방식으로, 영문/숫자는 1바이트, 한글은 대부분 3바이트를 차지합니다.
        </TabItem>
         <TabItem label="EUC-KR">
             완성형 한글을 표현하기 위해 개발된 인코딩 방식입니다. 한글 한 글자를 2바이트로 표현합니다.  과거에 많이 사용되었으나,  UTF-8의 등장으로 사용 빈도가 줄었습니다.
        </TabItem>
         <TabItem label="CP949">
              마이크로소프트에서 개발한 EUC-KR의 확장형입니다.  EUC-KR에서 표현하지 못하는 일부 한글(옛한글, 확장 한자 등)을 추가로 지원합니다.
        </TabItem>
    </Tabs>
    <Code lang="python" title="Python에서의 인코딩 확인">
```python
text = "한글"
print(len(text.encode('utf-8')))  # 6 (UTF-8) - 한글 한 글자당 3바이트, 총 2글자
print(len(text.encode('euc-kr'))) # 4 (EUC-KR) - 한글 한 글자당 2바이트, 총 2글자
```

  <Card title="조합형 vs 완성형: 한글 표현의 두 가지 방식" icon="puzzle">
    초창기 컴퓨터 환경에서는 한글을 표현하는 방식으로 **조합형**과 **완성형**이 경쟁했습니다.
    *   **조합형:** 초성, 중성, 종성을 각각 별개의 코드로 조합하여 한 글자를 표현하는 방식입니다.  이론적으로는 모든 한글 표현이 가능하지만 구현이 복잡하고 호환성 문제가 있었습니다.
    *   **완성형:** 현대 한글에서 사용되는 모든 글자(2,350자 또는 11,172자)에 각각 고유한 코드를 부여하는 방식입니다.  구현이 비교적 간단하고 호환성이 좋지만,  표현할 수 없는 글자가 존재한다는 단점이 있습니다. (옛한글 등)

    현재는 대부분의 시스템에서 **완성형** (특히 UTF-8)을 사용하고 있으며, 조합형은 거의 사용되지 않습니다.

  </Card>

  <Card title="한글 입력 시스템 (IME): 생각을 문자로" icon="keyboard">
    **IME (Input Method Editor)** 는 사용자가 키보드를 통해 한글을 입력할 수 있도록 도와주는 소프트웨어입니다.  한글은 초성, 중성, 종성이 결합하여 하나의 음절을 이루는 독특한 구조를 가지고 있기 때문에, IME는 이러한 조합 과정을 처리하는 핵심적인 역할을 합니다.

    *   **두벌식:**  현재 가장 널리 사용되는 한글 자판 배열입니다.  왼쪽에는 자음, 오른쪽에는 모음을 배치하여 효율적인 입력이 가능합니다.
    *   **세벌식:**  초성, 중성, 종성을 각각 다른 위치에 배치하여 더 빠른 입력 속도를 제공하는 자판 배열입니다.  숙련된 사용자에게는 매우 효율적이지만,  배우기 어렵다는 단점이 있습니다.

    IME는 사용자가 입력한 키를 조합하여 한글 음절을 생성하고,  이를 화면에 표시하는 복잡한 과정을 수행합니다.
  </Card>
  <Card title="프로그래밍과 한글: 개발자를 위한 안내" icon="code">
    대부분의 현대 프로그래밍 언어는 유니코드(Unicode)를 지원하므로, 한글을 직접 처리할 수 있습니다. 그러나 한글 처리에 특화된 기능이나 라이브러리를 사용하면 더욱 효율적인 개발이 가능합니다.

    *   **문자열 처리:**  한글 문자열의 길이 계산, 부분 문자열 추출, 검색, 치환 등 다양한 작업을 수행할 수 있습니다.  특히, 한글은 음절 단위로 처리해야 하는 경우가 많으므로,  이에 대한 이해가 필요합니다.
    *   **정규 표현식:**  한글을 포함하는 텍스트에서 특정 패턴을 찾거나 추출하는 데 사용됩니다.  한글 자모, 음절, 단어 등을 정확하게 매칭하는 정규 표현식을 작성하는 것이 중요합니다.
    *   **자연어 처리 (NLP):**  한글 텍스트를 분석하고 이해하는 기술입니다.  형태소 분석, 품사 태깅, 개체명 인식, 감성 분석 등 다양한 NLP 기술이 한글 처리에 활용됩니다.

    <Code lang="python" title="Python에서의 한글 문자열 처리">
```python
text = "안녕하세요, 한글입니다."
print(text[0:5])  # "안녕하세" (부분 문자열)
print("한글" in text)  # True (문자열 포함 여부)

import re
pattern = r"[가-힣]+" # 한글 음절 패턴
matches = re.findall(pattern, text)
print(matches) # ['안녕하세요', '한글입니다']
```
    </Code>

  <Card title="한글 폰트: 디지털 글꼴의 미학" icon="type">
    컴퓨터 화면이나 인쇄물에 한글을 표시하기 위해서는 **폰트**가 필요합니다. 한글 폰트는 글자의 모양(글리프)과 관련된 정보를 담고 있는 파일입니다.

    *  **TrueType (.ttf):**  가장 널리 사용되는 폰트 형식 중 하나입니다.  확대/축소 시에도 깨끗한 외곽선을 유지하는 벡터 기반 폰트입니다.
    *   **OpenType (.otf):**  TrueType의 확장된 형식으로,  더 다양한 기능과 고급 타이포그래피 기능을 지원합니다.
    *   **웹 폰트 (WOFF, WOFF2):**  웹 페이지에서 사용하기 위해 최적화된 폰트 형식입니다.  파일 크기를 줄여 로딩 속도를 향상시킵니다.

    한글 폰트는 디자인, 가독성, 용도 등에 따라 다양한 종류가 있으며,  폰트 렌더링 엔진은 폰트 파일에 저장된 정보를 바탕으로 화면에 글자를 그려냅니다.

## 더 깊이 알아보기: 유니코드와 한글

한글은 **유니코드(Unicode)** 표준의 일부로, 전 세계 모든 문자를 컴퓨터에서 일관되게 표현하고 처리하기 위한 국제적인 문자 코드 시스템입니다. 한글은 크게 세 영역으로 나뉩니다.

*   **한글 자모 (Hangul Jamo):**  `U+1100` ~ `U+11FF` (초성, 중성, 종성 자모)
*   **한글 호환 자모 (Hangul Compatibility Jamo):** `U+3130`~`U+318F` (호환용 자모 - 주로 다른 코드 페이지와의 호환성을 위해 존재.)
*   **한글 음절 (Hangul Syllables):**  `U+AC00` ~ `U+D7A3` (현대 한글에서 사용되는 모든 음절)

이러한 유니코드 할당 덕분에, 한글은 전 세계 모든 컴퓨터 시스템에서 문제없이 표현되고 처리될 수 있습니다.
