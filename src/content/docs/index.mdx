## 한글의 디지털 표현: 핵심 기술

<CardGrid stagger>
  <Card title="1. 문자 인코딩: 한글의 디지털 옷" icon="globe">
    컴퓨터는 모든 정보를 0과 1(이진수)로 표현합니다. 한글을 포함한 모든 문자는 숫자로 변환되어야 컴퓨터가 이해할 수 있습니다. 이 변환 과정을 **문자 인코딩**이라고 합니다.
    <Tabs>
        <TabItem label="UTF-8">
          가장 널리 쓰이는 국제 표준 인코딩입니다.  영문/숫자는 1바이트, 한글은 대부분 3바이트로 표현합니다 (가변 길이).  웹에서 주로 사용됩니다.
        </TabItem>
        <TabItem label="EUC-KR">
          과거에 많이 쓰였던 완성형 한글 인코딩입니다.  한글 한 글자를 2바이트로 표현합니다.  UTF-8 등장 이후 사용 빈도가 줄었습니다.
        </TabItem>
        <TabItem label="CP949">
          EUC-KR의 확장형으로, 마이크로소프트에서 개발했습니다. EUC-KR에서 표현하지 못하는 일부 글자(옛한글 등)를 추가로 지원합니다.
        </TabItem>
    </Tabs>
    **Python 예시 (간단 설명):**
    ```
    text = "한글"
    len(text.encode('utf-8'))  # 결과: 6 (3바이트 * 2글자)
    len(text.encode('euc-kr')) # 결과: 4 (2바이트 * 2글자)
    ```

  </Card>

  <Card title="2. 조합형 vs 완성형: 한글 표현 방식" icon="puzzle">
    초창기 컴퓨터에서 한글을 표현하는 두 가지 주요 방식은 조합형과 완성형이었습니다.
    *   **조합형:** 초성, 중성, 종성을 각각 별개의 코드로 조합하여 한 글자를 만듭니다. 이론적으로 모든 한글 표현이 가능하지만, 구현이 복잡하고 호환성 문제가 있었습니다.
    *   **완성형:** 자주 쓰이는 한글 글자(2,350자 또는 11,172자)에 미리 코드를 할당하는 방식입니다. 구현이 쉽고 호환성이 좋지만, 모든 한글을 표현할 수는 없습니다 (옛한글 등).

    현재는 대부분 **완성형**(UTF-8 등)을 사용합니다.
  </Card>

  <Card title="3. 한글 입력 시스템 (IME): 생각을 문자로" icon="keyboard">
    **IME (Input Method Editor)**는 사용자가 키보드로 한글을 입력하도록 돕는 소프트웨어입니다.  한글은 초성, 중성, 종성이 결합되어야 하므로, IME가 이 조합 과정을 처리합니다.

    *   **두벌식:** 가장 일반적인 한글 자판 배열. 왼쪽은 자음, 오른쪽은 모음.
    *   **세벌식:** 초성, 중성, 종성을 각각 다른 위치에 배치. 숙련되면 빠르지만 배우기 어려움.
  </Card>

  <Card title="4. 프로그래밍과 한글: 개발자를 위한 팁" icon="code">
    대부분의 프로그래밍 언어는 유니코드(Unicode)를 지원하여 한글을 직접 처리할 수 있습니다.
    *   **문자열 처리:** 한글 문자열 길이 계산, 부분 추출, 검색, 치환 등을 할 수 있습니다. 음절 단위 처리에 유의해야 합니다.
    *   **정규 표현식:** 한글 포함 텍스트에서 특정 패턴을 찾거나 추출할 때 유용합니다. `[가-힣]`과 같은 패턴을 사용합니다.
      * 예시: `[가-힣]+` (한글 음절이 1개 이상 연속되는 패턴)
    *   **자연어 처리 (NLP):** 한글 텍스트 분석(형태소 분석, 품사 태깅 등)에 사용됩니다.

     **Python 예시 (간단 설명):**
    ```python
    text = "안녕하세요, 한글입니다."
    text[0:5]  # "안녕하세" (부분 문자열)
    "한글" in text  # True (포함 여부)

    import re
    re.findall(r"[가-힣]+", text)  # ['안녕하세요', '한글입니다']
    ```
  </Card>

  <Card title="5. 한글 폰트: 디지털 글꼴" icon="type">
     컴퓨터 화면이나 인쇄물에 한글을 표시하려면 **폰트** 파일이 필요합니다. 폰트는 글자 모양(글리프) 정보를 담고 있습니다.

    *   **TrueType (.ttf):** 널리 쓰이는 벡터 기반 폰트. 확대/축소해도 깨지지 않음.
    *   **OpenType (.otf):** TrueType의 확장형. 더 많은 기능과 고급 타이포그래피 지원.
    *   **웹 폰트 (WOFF, WOFF2):** 웹 페이지용으로 최적화된 폰트. 파일 크기가 작음.
  </Card>
</CardGrid>

## 더 깊이 알아보기: 유니코드와 한글

한글은 **유니코드(Unicode)**라는 국제 표준 문자 코드 시스템의 일부입니다. 유니코드는 전 세계 모든 문자를 컴퓨터에서 일관되게 표현하기 위한 것입니다.  한글은 유니코드에서 다음 영역에 할당되어 있습니다.

*   **한글 자모 (Hangul Jamo):** `U+1100` ~ `U+11FF` (초성, 중성, 종성)
*   **한글 호환 자모 (Hangul Compatibility Jamo):** `U+3130`~`U+318F` (다른 코드 페이지와 호환을 위해 존재)
*   **한글 음절 (Hangul Syllables):** `U+AC00` ~ `U+D7A3` (현대 한글 음절)
