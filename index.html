``` 주요 변경 사항: 1. **Gemini API Key 입력 및 저장:** * HTML에 API 키 입력 필드(`api-key`)와 저장 버튼(`save-api-key`)을 추가했습니다. * `saveApiKeyButton` 클릭 이벤트 핸들러를 추가하여, 입력된 API 키를 `localStorage`에 저장합니다. * `loadApiKey()` 함수를 추가하여 페이지 로드 시 `localStorage`에서 API 키를 불러옵니다. * `callGeminiAPI`함수에서 API호출 전에, `loadApiKey()`를 호출하여 저장된 키를 가져옵니다. 2. **Gemini API 호출 함수 (`callGeminiAPI`) 분리:** * Gemini API 호출 로직을 별도의 `callGeminiAPI` 함수로 분리하여 코드를 더 깔끔하게 만들었습니다. * API 키가 없으면 `null`을 리턴하고, Gemini API를 호출하지 않습니다. * API 호출 실패 시, 에러 메시지를 사용자에게 보여주고, 기본 응답 문자열을 반환합니다. 3. **기본 응답 (Fallback):** * `sendMessage` 함수에서 Gemini API 응답이 `null` (API 키가 없거나 호출 실패)일 경우, 캐릭터별 기본 응답을 표시하도록 했습니다. * `defaultResponses` 객체에 캐릭터별 기본 대사를 저장. 4. **로딩 메시지:** * HTML에 `loading-message` 요소를 추가하고, CSS로 스타일을 지정했습니다. * `showLoading(true/false)` 함수를 추가하여 로딩 메시지를 표시하거나 숨깁니다. * `sendMessage` 함수에서 API 호출 전/후에 `showLoading`을 호출하여 로딩 상태를 표시합니다. 5. **`async`/`await`:** `updateCharacterDisplay`, `sendMessage`, `startPractice`, `startGame` 함수를 `async` 함수로 변경하고, `fetch` 호출 시 `await`를 사용하여 비동기 처리를 올바르게 합니다. 6. **`PRACTICE_CONTENT` , `GAME_CONTENT` 객체 사용:** * 연습/게임 컨텐츠를 함수가 아닌 객체에 저장, 함수 호출을 제거하여 코드를 간단하게 만듦. 7. **`fetch` 제거, `data` 직접 할당:** `startPractice`와 `startGame`에서 `fetch`를 사용하지 않고, `PRACTICE_CONTENT`와 `GAME_CONTENT` 객체에서 직접 데이터를 가져와 할당합니다.
